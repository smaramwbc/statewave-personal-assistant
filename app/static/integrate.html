Concept

How it works

Three calls wrap every conversation turn. That's the entire integration surface.

1
GET context — before the LLM call

POST to /v1/context with the user ID and current message. Statewave returns an assembled_context string — the highest-ranked facts about this user that fit within your token budget.

2
Inject into system prompt — call the LLM

Append the context string to your system prompt. The model sees what it knows about this user without you managing retrieval, ranking, or chunking.

3
RECORD episode — after the LLM response

POST the user message + assistant reply to /v1/episodes. Statewave extracts structured memory facts asynchronously — preferences, open issues, profile facts — ready for the next session.

Token budget: Statewave never returns more context than you allow. Set max_tokens: 800 and it fits the most relevant facts into 800 tokens — so your LLM context window stays predictable even after 100 sessions.

Pattern 1

Plain async wrapper

Works with FastAPI, Django Async, raw asyncio — any Python async context. Replace your existing openai.chat() call with memory_chat().

python — before 
from openai import AsyncOpenAI

client = AsyncOpenAI()

async def chat(user_id: str, message: str) -> str:
    completion = await client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": message}],
    )
    return completion.choices[0].message.content
python — after (3 lines changed) 
from statewave_integration import memory_chat

async def chat(user_id: str, message: str) -> str:
    return await memory_chat(user_id=user_id, message=message)
    # ↑ fetches memory, calls LLM, records episode — all in one call
python — with custom system prompt 
reply = await memory_chat(
    user_id="alice",
    message="What was I debugging last week?",
    system_prompt="You are a senior engineer helping with code reviews.",
    model="gpt-4o",
)
Pattern 2

FastAPI middleware

Zero changes to your existing route handlers. The middleware intercepts every POST to /chat, fetches memory before your handler runs, and records the episode after it responds.

python — main.py (1 line added) 
from fastapi import FastAPI
from statewave_integration import StatewaveMemoryMiddleware

app = FastAPI()
app.add_middleware(StatewaveMemoryMiddleware)

@app.post("/chat")
async def chat(req: ChatRequest, request: Request):
    # memory_context is injected into request.state by the middleware
    context = getattr(request.state, "memory_context", "")
    reply = await llm_call(req.message, system_context=context)
    return {"response": reply}
Custom path: By default the middleware watches /chat. Pass chat_path="/api/v1/messages" to add_middleware() to match your route.
Pattern 3

LangChain chain step

Drop statewave_memory_step at the front of any LCEL chain. It fetches memory and injects it as a memory_context variable your prompt template can reference.

python 
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from statewave_integration import statewave_memory_step

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.\n\n{memory_context}"),
    ("human",  "{message}"),
])

chain = statewave_memory_step | prompt | ChatOpenAI() | StrOutputParser()

reply = await chain.ainvoke({
    "user_id": "alice",
    "message": "What was I building last week?",
})
Reference

Statewave API endpoints

The two endpoints you'll call directly (the integration file wraps these).

POST /v1/context — fetch ranked memory 
# Request
{
  "subject_id": "alice",
  "task":       "What was I debugging last week?",  // optional — improves ranking
  "max_tokens": 800
}

# Response
{
  "assembled_context": "User is a senior backend engineer at Acme...",
  "token_estimate":    312,
  "facts": [
    { "kind": "profile_fact", "content": "...", "score": 0.94 },
    ...
  ]
}
POST /v1/episodes — record a turn 
{
  "subject_id": "alice",
  "source":     "chat",
  "type":       "conversation",
  "payload": {
    "messages": [
      { "role": "user",      "content": "What was I debugging?" },
      { "role": "assistant", "content": "Last week you were investigating..." }
    ]
  },
  "metadata": {}
}
EndpointWhen to callFatal if fails?
POST /v1/contextBefore every LLM callNo — fall back to empty context
POST /v1/episodesAfter every LLM responseNo — fire and forget
POST /v1/memories/compileOn-demand or scheduledNo
GET /v1/memories/searchInspect / debugNo
